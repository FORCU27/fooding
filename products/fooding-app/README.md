# Fooding App (매장용)

| Field | Detail |
| --- | --- |
| **Primary Target** | 사장님/직원(홀 매니저, 리워드 담당자) |
| **Device** | iOS/Android (React Native) |
| **Role** | POS와 CEO 사이를 잇는 모바일 운영 허브 |

## Value Proposition
- 매장 실무진이 **언제 어디서나 웨이팅/예약 현황**을 확인하고 제어할 수 있습니다.
- 직원이 카운터가 아닌 자리에서도 **포인트 적립, 쿠폰 발급, 응대 메모**를 처리합니다.
- 키오스크/QR 주문 상태를 실시간으로 받아보며 고객 응대를 즉시 조정합니다.

## Primary Workflows
1. **웨이팅 관리**: 원격 등록 승인, 순번 조정, SMS/앱 알림 트리거, 예상 대기시간 수정.
2. **예약 관리**: 사전 주문 확인, 좌석 배정, 보증금 확인, 노쇼 처리.
3. **리워드/쿠폰**: QR 스캔/전화번호 검색으로 적립, 포인트/쿠폰 차감, 방문 기록 확인.
4. **직원 커뮤니케이션**: 교대 로그, 공지, 빠른 이슈 리포트.
5. **키오스크 연계**: 주문 흐름/준비 상태 확인, 준비 완료 알림 전송.

## KPIs
- 모바일에서 처리한 웨이팅/예약 건수, 평균 응답 시간, 현장 승인율.
- 리워드 적립 성공률, 쿠폰 사용 처리 시간, 앱 사용 직원 수(DAU).
- 고객 알림 정확도, 노쇼 감소율.

## Dependencies
- POS/Reservation/Waiting/Rewards API, Notification, Staff auth 서비스.
- SSE 기반 웨이팅 스트림, Kafka 이벤트 기반 상태 업데이트.
- 모바일 빌드 파이프라인: React Native + Storybook 컴포넌트 라이브러리.

## Current Status & Plans
- ✅ 웨이팅 매장관리, CEO 앱/Electron·RN 셋업, 리워드 매장관리 앱 기능 통합.
- 🔄 직원 권한 세분화, 교대 자동 기록, 스캔 오류 처리 UX 개선.
- 📌 향후엔 POS 하드웨어 알림과 연계한 **하이브리드 핸드오프** 기능을 지원합니다.
